1. EXECUTIVE SUMMARY
--------------------------------------------------------------------------------
This project implements an advanced Network Intelligence System designed to model, analyze, and secure the "Phantom Troupe" network structure. To address the challenge constraints—specifically the mandate that the Minimum Spanning Tree (MST) must not collapse due to implicit rule violations—we engineered a multi-layered solution combining Kruskal's Algorithm with a custom heuristic Anomaly Detection AI.

2. CORE ALGORITHM: PROTECTED KRUSKAL'S MST
--------------------------------------------------------------------------------
The foundational MST is built using Kruskal’s Algorithm, backed by a Disjoint Set Union (DSU) data structure featuring both Path Compression and Union by Rank.
- Complexity: The time complexity is bounded by O(E log E) for sorting the edges, and O(α(V)) for the union-find operations, making it highly optimal.
- Dynamic Recomputation: Given the dynamic nature of the problem, the MST is recomputed instantaneously upon any graph mutation (Add/Remove/Update).

3. ADVANCED ANOMALY DETECTION ENGINE
--------------------------------------------------------------------------------
The primary challenge warns of "implicit constraints" and "cursed edges." A naive approach (e.g., just filtering negative weights) is insufficient. We implemented a continuous, multi-dimensional validation system:

A. Baseline Validation:
   - Weights <= 0: Flagged as structurally impossible (Suspicion: 100%).
   - Weights > 20: Flagged as exceeding natural game limits (Suspicion: 100%).
   - Self-Loops: Paradoxical connections are instantly rejected.

B. Geometric Stability Check (Triangle Inequality):
   - The most advanced layer of our detection system checks for topological paradoxes. In a stable metric space, the direct connection between two nodes should logically be less than or equal to the shortest detour through an adjacent third node.
   - For every edge (A, C), we calculate `minDetour = Weight(A, B) + Weight(B, C)`.
   - If `Weight(A, C) > minDetour`, the edge violates the geometric stability of the web.
   - Suspicion Scoring: The severity of the violation is calculated proportionally: `Suspicion = (Weight / minDetour) * 50`. 

C. Quarantine Protocol:
   - Edges with a Suspicion Score >= 80% are designated as "Cursed Edges".
   - The MST Engine strictly filters out ALL Cursed Edges *before* executing Kruskal's algorithm, absolutely guaranteeing that the resulting tree is pure and mathematically stable.

4. VISUALIZATION & PHYSICS SIMULATION
--------------------------------------------------------------------------------
Rather than using heavy external libraries (like D3.js or Vis.js), we built a proprietary Force-Directed Graph Engine using HTML5 Canvas and Vanilla JavaScript.
- Repulsion Forces: Nodes act as charged particles, constantly repelling each other (Coulomb's Law approximation).
- Spring Attraction: Edges act as springs, pulling connected nodes together based on Hooke's Law, modified by edge weight.
- Gravity & Dampening: A central gravitational pull keeps the network centered, while kinetic dampening prevents infinite oscillation.
- The result is an "Alive Web" that breathes and reacts to structural changes in real-time.

5. ENTERPRISE UI & FEATURES
--------------------------------------------------------------------------------
- Algorithmic Animation: Users can trigger a step-by-step visual animation of Kruskal's algorithm building the MST.
- State Management (Undo/Redo): Implemented a robust state-history stack allowing the user to seamlessly roll back structural changes.
- Import/Export: Network states can be serialized to JSON and exported/imported for offline analysis.
- Cyberpunk Design System: A highly stylized, immersive "Hacker Terminal" UI utilizing glassmorphism, scanline overlays, and neon typography to match the lore of the Phantom Troupe.
